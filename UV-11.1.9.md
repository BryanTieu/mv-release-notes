UNV-1492 UniVerse BASIC - When doing mathematical operations using
numbers greater that 15 digits, unwanted rounding may have occurred. At
this release, the BASIC compiler option STRING.MATH has been enhanced to
also handle the INT, ABS, NEG, and MOD functions.

        Additionally, at previous releases, formatting problems
        existed for certain format specifications when the data

        the FMT function could result in precision being lost. This

UNV-2320 UniObjects - If UniObjects connections were made to a UniVerse

UNV-2702 UniVerse - The fixtool utility has been modified to handle a
situation where free buffers are linked to both a primary group and to
the free chain. At prior releases, this situation may have caused
fixtool to dump core. This problem has been fixed. Note, after running
fixtool on a file in this condition, a RESIZE operation may be needed to
correct an issue with orphaned buffers.

SoapSetRequestHeader() function did not allow the Content-Type header to
be changed. Now, the SoapCreateRequest() function uses the default

UNV-4915 U2 Data Replication -- At this release, U2 Data Replication has
been enhanced to allow wildcard patterns in the EXCLUDED_FILE phrase.
This is helpful if your application creates temporary files that you may
not know the exact name of in advance.

        In the repconfig or repacct.def files, define the pattern
        for the files you want to exclude using the EXCLUDED_FILE
        phrase. 

        EXCLUDED_FILE={DICT | DATA} [<wildcard>]<filename>
        [<wildcard>]

        in which <wildcard> is "..."

        If wildcard exists in the EXCLUDED_FILE phrase, it
        describes an excluded pattern rather than an excluded
        file. For example, the phrase "EXCLUDED_FILE=
        ...WORK.FILE..." specifies any file containing "WORK.FILE"
        to be excluded.

        "EXCLUDED_FILE=WORK..." specifies an excluded pattern for
        WORK.*.

        'EXCLUDED_FILE=WORK.FILE,..." specifies an excluded pattern
        that matches all subfiles of the level file "WORK.FILE"

        In account-level replication, the RESERVED_FILE_SPACE
        parameter defines the number of dynamic objects for both
        included and excluded objects. Each dynamic excluded
        object created for a file will consume one slot in the
        dynamic object table.

        The default repacct.def file now contains the following
        excluded file phrase:

        EXCLUDED_FILE=VI.TMP...

UNV-5533 UniVerse - Before this release, UniVerse processes could not
attach memory mapped files in excess of 2GB on Windows platforms. To
solve this problem, UniVerse now links dlls and executables with the

memory mapped files on Windows 2003 or earlier, add the '/3GB' option to
the Windows boot.ini file. This reduces the Windows kernel addressing
from 2GB to 1GB, but expands the user processing address from 2GB to
3GB.

        On systems after Windows 2003, used the 'BCDEDIT /set'
        command and set the 'increaseuserva' boot entry option to
        a value between 2048 (2GB) and 3072 (3 GB). For example,
        'bcdedit /set increaseuserva 3072'.

        For more information, see http://msdn.microsoft.com/en-us/
        library/bb613473.aspx.

UNV-4984 EDA -- The EDA SQL Server Driver for UNIX and Linux has been
added at this release. The EDA SQL Server Driver for UNIX and Linux is
used to provide access for UniVerse installations running on UNIX or
Linux to Microsoft SQL Server running on Windows platforms. The EDA SQL
Server Driver for UNIX and Linux is a dynamic-loading library. It uses
unixODBC as the ODBC data source manager on UNIX and Linux platforms,
and uses a third-party ODBC driver, such as the Easysoft ODBC driver, to
communicate with Microsoft SQL Server. The EDA SQL Server Driver for
UNIX and Linux is available on AIX, HP Itanium, Solaris SPARC, and Linux
platforms.

        Install unixODBC and Third-party ODBC Driver
        --------------------------------------------

        Install the ODBC data source manager, unixODBC, and the
        third-party ODBC driver on the same machine where
        UniVerse is installed. If the third-party ODBC driver,
        such as Easysoft SQL Server, includes unixODBC, you
        only need to install the ODBC driver.

        Set Up the EDA Environment
        --------------------------

        On UNIX platforms, execute the operating system-level
        command edasetup.sh to set up the EDA environment. This
        command prompts you for information and generates the
        edaconfig file in the $UVHOME accoount. The edaconfig
        file contains the following information:

            DRIVER=ODBC
            ODBCDRVPATH=/usr/local/easysoft
            LOGLEVEL=0

        You can change the LOGLEVEL to 1 or 2. The higher the
        log level, the more information is captured.

        Set Up Connection to the External Database
        ------------------------------------------

        To connect to an external database, you must set up
        a connection using unixODBC. To do this, add a data
        source and an ODBC driver. You can add a system data
        source which is available to anyone who logs on to
        the UNIX machine, or a user data source which is
        available only to the user for which it was created.
        The installation process adds the system data source
        to /etc/odbc.ini. The user data source is added to
        $HOME/.odbc.ini.

        The following example illustrates an entry for a
        data source:

        [SQL2005]
        Driver = Easysoft ODBC-SQL Server
        Description  = SQL Server DSN created during installation
        Server = jyaosql
        Port  = 
        User  = jyao
        Password = 1234
        Language = English
        Database = DEVDB
        Logging  = 0
        LogFile  = 
        QuotedId = Yes
        AnsiNPW  = Yes
        Mars_Connection = Yes

        The next example illustrates an entry for an ODBC driver.
        The system driver entry is located in /etc/odbcinist.ini.
        The user driver is located in $HOME/.odbc.ini:

        [Easysoft ODBC-SQL Server]
        Driver = /usr/local/easysoft/sqlserver/lib/libessqlsrv.a
        Setup = /usr/local/easysoft/sqlserver/lib/libessqlsrvS.a
        Trace = yes
        TraceFile = /tmp/easysoft_odbc
        MARS_Connection = Yes
        Threading = 0
        FileUsage = 1
        DontDLClose = 1
        UsageCount = 2

        Set Up the ODBC Dynamic Loading Library Path
        --------------------------------------------

        To load the ODBC driver's libraries, you must set up the
        ODBC dynamic loading library path.

        For AIX platforms, add the ODBC driver's library path
        to LIBPATH.

        For HP Itanium and Linux platforms, add the ODBC driver's
        library path to LD_LIBRARY_PATH.

        You also need to set up an environment variable for the
        ODBCPATH. To set up the environment variable, use the
        following command:

            setenv ODBCPATH /usr/loca/easysoft/unixODBC

        After setting the library path, you must restart
        unirpcd.

        Create the EDA Data Source
        --------------------------

        Create the EDA data source using the EDA Schema
        Manager. Make sure you use the data source you
        specified when installing the third-party ODBC driver.

        Data Type Mapping
        -----------------

UNV-4842 UniVerse -- At UniVerse 11.1, some enhancements have been made
to the UniVerse shutdown process. When using the standard "uv -admin
-stop" command, the shutdown process will behave in the same manner as
UniVerse 10.3, with one exception. At UniVerse 11.1, when the shutdown
process has been initiated, a flag is set in shared memory which will
inhibit UniVerse logins while the shutdown is in process. Any user
attempting to log in will receive a warning message indicating the
shutdown is in process and logins are not allowed.

        The new options available at UniVerse 11.1.1 are as
        follows:

            uv -admin -stop -force[-now]

        As with the normal shutdown, a SIGTERM signal will be
        sent to all user processes to terminate these processes.
        During a normal shutdown, if user processes remain, the
        shutdown process will exit and display a message
        indicating the shutdown did not complete. When using the
        "-force" option, if an processes remain after the
        SIGTERM, a SIGKILL signal will be sent to the remaining
        processes. If the "-now" option is not used, the shutdown
        process will wait for a short period of time before
        issuing the SIGKILL signal. If the "-now" option is used,
        the SIGKILL will be sent immediately to those processes
        which were not terminated by the SIGTERM signal.

            uv -admin -stop -nouser

        The "-nouser" option results in the shutdown process
        terminating with a message indicating the shutdown
        process did not take place if there are any active user
        processes when the command was issued. 

        As noted previously, the shutdown process  will not set
        a flag in shared memory indicating a shutdown is in
        process so that new logins can be inhibited. The setting
        of this flag is down through the stopuvsmm command.

        The following options have been added to the stopuvsmm
        command:

            stopuvsmm [-P] [-S] [-F] [0 | 1 |2] [:<pid>]

        The -P option will display the current system shutdown
        state. The shutdown state can be one of the following:

            0 - systemrunning normally
            1 - shutdown in process
            2 - shutdown pending

        The -S option can be used to set one of the states
        mentioned above. The -S option will check the current
        shutdown state and will exit with an error message if
        another shutdown process is active. The -F option will
        set the shutdown state to the specified value without
        checking if another process is active.

        The shutdown state of 2 can be used by an administrator
        to inhibit new logins while the system is prepared for
        shutdown.

        Typically, the <pid> option would not be used and the
        pid of the current process executing the stopuvsmm
        command will be registered in the system as the active
        shutdown process.

        been added to the uvrestore command. The -convtype1

        executing the uvbackup command. When restoring the
