## General Hotfix information 

This release note contains a cumulative summary of all the Hotfixes made available for a specific Generally Available (GA) product version. Please browse this document to find the Hotfix release that fits your specific needs. 

Hotfixes are identified as "Controlled Releases" within Rocket Business Connect (RBC). Hotfixes are only visible by searching for the specific version and build number in the Controlled Release field. 

To obtain a specific Hotfix: 

1. Log into the RBC system and modify your existing license. 

2. Click **Edit** next to your existing product and then click **Search** to view the list of GA product releases. 

3. After the product availability matrix displays, enter in the Hotfix version number (which includes the product version and build number, for example 11.3.1.6005) in the **Controlled Release** field just above the table of product releases, then click **Search** . 

The Hotfix release will display on the product matrix and can be selected for ordering and download. If you have questions about RBC, please contact us at U2Support@rocketsoftware.com. 

**Note:** Hotfix releases are intended to be short-term solutions and have had limited testing in order to be made available quickly. All Hotfix changes will be included in the next GA release, and will be fully tested by Rocket’s Quality Assurance (QA) team across all supported platforms and environments. It is strongly recommended that after taking a Hotfix customers upgrade to the next available GA release to ensure the highest quality experience. 

For questions, please contact support at U2Support@rocketsoftware.com .

|UNV-24061|Beginning with this release, support has been added for query parameters to be passed<br>within the URL for the HTTP`PUT`and`DELETE`methods, in addition to the HTTP`GET`<br>and`POST`methods that already support this function.<br>For example, http://webserver/Customer?ID=xxxx&NAME=smith. For more information<br>about these query parameters, see the_UniVerse BASIC Extensions guide_.|
|UNV-24974|Windows only. Beginning with this release, the design of the BASIC`INPUT`statement<br>for a uvtelnet session has been modified. The modification has significantly improved<br>the performance of the`INPUT`statement when used for transferring very large<br>amounts of data. The testing used input strings of multi-thousands of characters. This<br>issue has been resolved.|
|UNV-25129|Windows only. Prior to this release, certain uvsmm errors could be repeatedly written<br>to the`uvsmm.errlog file`. The result would be a very large`uvsmm.errlog`file<br>which could either cause disk space problems and/or problems attempting to restart<br>UniVerse. In the current release, repeating errors will be written a maximum of 10 times,<br>thus resolving this issue.|
|UNV-25256|Linux only. Prior to this release, a UniVerse process may have become hung on a`FUTEX`<br>lock while performing a`U2free`operation. This was a timing issue related to the<br>process receiving a signal during the`U2free`operation. This issue has been resolved in<br>the current release.|
|UNV-26152|Beginning with this release, the_ALLOWNULLS_uvconfig parameter has been added. The<br>_ALLOWNULLS_parameter can be used to control the ability to write null keys to UniVerse<br>files. Setting this parameter to 1 (default) allows null keys to be written. When set to 0,<br>attempting to`WRITE`a null key will cause a fatal error and terminate the program, or<br>take the`ON ERROR`clause if it exists.|
|UNV-27484|Starting with this release, the`uvdiag`script has been updated to version 5.2.3 for UNIX<br>and Linux platforms.|
|UNV-27621|Prior to this release, a problem existed when using`DATA`statements to provide input<br>to`INPUT`statements in subroutines called from UniObjects. A mismatch between<br>the number of`INPUT`and`DATA`statements was treated as a fatal error and the UO<br>subroutine would stop responding. Restarting the uvapi_slave process on the server<br>was the only way to resolve the issue. This issue has been resolved.|
|UNV-28426|Beginning with this release, the`INPLACE`option of`RESIZE`is no longer a valid option.<br>A syntax error will be produced at the current release if the`INPLACE`option is used.|
