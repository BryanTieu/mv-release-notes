## General Hotfix information 

This release note contains a cumulative summary of all the Hotfixes made available for a specific Generally Available (GA) product version. Please browse this document to find the Hotfix release that fits your specific needs. 

Hotfixes are identified as "Controlled Releases" within Rocket Business Connect (RBC). Hotfixes are only visible by searching for the specific version and build number in the Controlled Release field. 

To obtain a specific Hotfix: 

1. Log into the RBC system and modify your existing license. 

2. Click **Edit** next to your existing product and then click **Search** to view the list of GA product releases. 

3. After the product availability matrix displays, enter in the Hotfix version number (which includes the product version and build number, for example 11.3.1.6005) in the **Controlled Release** field just above the table of product releases, then click **Search** . 

The Hotfix release will display on the product matrix and can be selected for ordering and download. If you have questions about RBC, please contact us at U2Support@rocketsoftware.com. 

**Note:** Hotfix releases are intended to be short-term solutions and have had limited testing in order to be made available quickly. All Hotfix changes will be included in the next GA release, and will be fully tested by Rocket’s Quality Assurance (QA) team across all supported platforms and environments. It is strongly recommended that after taking a Hotfix customers upgrade to the next available GA release to ensure the highest quality experience. 

For questions, please contact support at U2Support@rocketsoftware.com .

## Rocket UniVerse Hotfix 11.3.2.7002 

## May 2020 

## Applicable platforms 

- AIX 

- Linux 

- Windows 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at support@rocketsoftware.com. 

## Description of the problem 

|**Issue**<br>**number**|**Description**|
|---|---|
|UNV-27535|Prior to this release, a memory leak existed when using the`MQCONN`and`MQDISC`<br>BASIC functions in a Phantom process. This issue has been resolved.|
|UNV-27536|Prior to this release, BASIC programs using the`MQ`functionality could randomly hang,<br>When this occurred, the process typically would need to be terminated because the<br>break key did not work. This issue has been resolved.|
|UNV-27561|Prior to this release, the Transaction Logging daemon (uvlogd) would incorrectly start<br>the uvlogswtch process in the same process group. This could have resulted in multiple<br>uvlogswtch processes running at the same time, consuming cpu. This issue has been<br>resolved.|
|UNV-28465|Beginning with this release, the new TCL`REINIT.PYTHON`command has been added<br>to control the re-initialization of the Python environment. The available options to the<br>command are`ON`/`OFF`/`STATUS`. The default value is`ON`. The`STATUS`option shows<br>the current setting. Note that this was incorrectly included in the 11.3.2 release as being<br>fixed in that release.|
|UNV-28923|Beginning with this release, the performance of u2py has been improved when reading<br>and writing UniVerse files.|
|UNV-28974|In a previous release, changes were made to improve the performance of the<br>Replication dynamic object table (UNV-28186). This change altered the way the primary<br>file reference was assigned when a VOC pointer to the file existed in multiple account<br>level groups. This release restores the original behavior. The primary file reference will<br>be assigned to the account level group which appears first in the`repconfig`file.|
|UNV-29157|Windows only. Beginning with 11.3.2, the`SYSTEM(42)`BASIC function may have<br>caused the UniVerse session to terminate unexpectedly. This issue has been resolved.|
|UNV-29323|Beginning with 11.3.2, new BASIC functions were added for UTC Datetime and<br>Geospatial functionality. The opcodes used for these new functions had conflicts with<br>new BASIC functions added at 12.1.1. In this release, the opcodes used for the new UTC<br>Datetime and Geospatial functionality have been changed to eliminate the conflict. Any<br>BASIC code already using the new functions added at 11.3.2 will require recompilation<br>with this release.|
|UNV-29379|The Replication protocol level should have been updated when 11.3.2 was released<br>due to the addition of Sequential I/O Replication at this release. The replication<br>protocol level is increased to 10010 at this release. Note, if using Replication with a<br>prior 11.3.2 release, all publishers and subscribers should be updated to this release for<br>compatibility.|

This would result in<br>errors being generated on the subscriber related to index file updates. This issue has<br>been resolved.|
