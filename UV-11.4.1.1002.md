## General Hotfix information 

This release notes document contains a cumulative summary of all the Hotfixes made available for a specific Generally Available (GA) product version. Please browse this document to find the Hotfix release that fits your specific needs. 

Hotfixes are identified as "Controlled Releases" within Rocket Business Connect (RBC). Hotfixes are only visible by searching for the specific version and build number in the Controlled Release field. 

To obtain a specific Hotfix: 

1. Log into the RBC system and modify your existing license. 

2. Click **Edit** next to your existing product, and then click **Search** to view the list of GA product releases. 

3. After the product availability matrix displays, enter the Hotfix version number (which includes the product version and build number, for example 11.3.1.6005) in the **Controlled Release** field just above the table of product releases. 

4. Click **Search** . 

The Hotfix release will display on the product matrix, and can be selected for ordering and download. If you have questions about RBC, please contact us at U2Support@rocketsoftware.com. 

**Note:** Hotfix releases are intended to be short-term solutions and have had limited testing in order to be made available quickly. All Hotfix changes will be included in the next GA release, and will be fully tested by Rocket’s Quality Assurance (QA) team across all supported platforms and environments. It is strongly recommended that after taking a Hotfix, customers upgrade to the next available GA release to ensure the highest quality experience. 

For questions, please contact support at U2Support@rocketsoftware.com.

## Rocket UniVerse Hotfix 11.4.1.1002 

## January 2024 

## Applicable platforms 

- AIX 

- Linux 

- Windows 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at support@rocketsoftware.com. 

## Description of the problem 

|**Issue**<br>**number**|**Description**|
|---|---|
|UNV-34543|Beginning with 11.3.5, the logic used to record the unirpcd startup activity in a<br>temporary log was incorrect. This resulted in the`startuv.log`containing incorrect<br>information related to the starting of the unirpcd daemon. This issue has been resolved.|
|UNV-34560|Beginning with 11.4.1, the uvdiag script failed to identify the`UVHOME`directory and<br>would abort without processing. This issue has been resolved.|
|UNV-34610|A fix introduced in 11.3.5.1002 and 11.4.1, related to transaction processing<br>(UNV-33817), had the negative side-effect of causing a memory leak in certain<br>situations. If the`READU`operation occurred outside of the transaction boundaries and<br>the files being updated did not contain an index, allocated memory was not released.<br>This could result in the uvsh process running out of memory and terminating if enough<br>transactions were processed. This issue has been resolved.|
|UNV-34611|AIX only. Beginning with 11.3.5.1001 and 11.4.1, a UO connection may have caused<br><defunct> processes to be listed in the process table while the UO connection was<br>active. This issue has been resolved in the current release.|
|UNV-34622|Beginning with 11.4.1, using the break key to enter the debugger on a running program<br>would terminate the process. After hitting the break key and entering 'D' to drop into<br>the debugger, the process would terminate unexpectedly and generate a core file. This<br>issue has been resolved in the current release.|
|UNV-34626|Beginning with 11.4.1, a "uv_repadmin reconfig" failure on the subscriber could<br>result in writes and deletes failing to be applied on the subscriber. If the reconfig<br>failure caused the previous configuration to be reloaded, subsequent updates to the<br>subscriber would be rejected with an error similar to:<br>`RW(1, 12124524):Excluded command(INSERT) is configured;`<br>`(LSN=7)Insert on FILE1('RECID') is prevented.`<br>This issue has been resolved in the current release.|
|UNV-34675|MVX only. Prior to this release, a memory allocation issue may have caused the<br>mvservice-worker process to terminate unexpectedly. The issue was related to MVX<br>recording file usage statistics for UniVerse files. This issue has been resolved in the<br>current release.|

## Rocket UniVerse Hotfix 11.4.1.1001 

## October 2023 

## Applicable platforms 

- AIX 

- Linux 

- Windows 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at support@rocketsoftware.com. 

## Description of the problem 

|**Issue**<br>**number**|**Description**|
|---|---|
|UNV-34294|Beginning in 11.3.4, the use of the PYTHONPATH environment variable was no longer<br>recognized by UniVerse. This issue has been resolved in the current release.|
|UNV-34347|Two changes have been made in this release related to using the BASIC`DELETE`<br>statement on a non-existent record in an EDA file.<br>Prior to this release, a`DELETE`operation on a non-existent record in an EDA file was<br>treated as a failure. This differed from the behavior on a standard UV file. The EDA file<br>behavior has been changed in this release to be consistent with that of a standard UV<br>file.<br>A change included in the 11.3.1.6023 release related to deleting records in an EDA file<br>introduced a potential problem. When deleting a non-existent record, an error message<br>similar to`Unable to unlink...`could be generated (and in certain circumstances<br>the process would terminate unexpectedly). This issue has been resolved in the current<br>release.|
|UNV-34375|Windows only. Beginning with 11.3.4, the BASIC`KEYIN()`function may have behaved<br>incorrectly under specific conditions. After quitting from a`Press any key....`<br>prompt, the execution of the`KEYIN()`function within that same session would get<br>interrupted based on the PAKTIME uvconfig setting. Program execution would continue<br>with the next statement even though no input had been provided. This issue has been<br>resolved in the current release.|
|UNV-34438|Prior to this release, the`USE.LEGACY.OPENSSL.ALGORITHM`command would fail<br>if the SSL_VERSION_PATH functionality was in use. The command would fail with an<br>error when attempting to turn it ON. This issue has been resolved in the current release.|
|UNV-34477|Prior to this release, the BASIC`RECORDLOCKED()`function returned incorrect results<br>when used with an EDA file. If the record was locked, the`RECORDLOCKED()`function<br>would incorrectly return a value of 0 indicating that no lock was set. This issue has been<br>resolved in this release.|
