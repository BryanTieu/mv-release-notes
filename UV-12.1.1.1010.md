## General Hotfix information 

This release note contains a cumulative summary of all the Hotfixes made available for a specific Generally Available (GA) product version. Please browse this document to find the Hotfix release that fits your specific needs. 

Hotfixes are identified as "Controlled Releases" within Rocket Business Connect (RBC). Hotfixes are only visible by searching for the specific version and build number in the Controlled Release field. 

To obtain a specific Hotfix: 

1. Log into the RBC system and modify your existing license. 

2. Click **Edit** next to your existing product and then click **Search** to view the list of GA product releases. 

3. After the product availability matrix displays, enter in the Hotfix version number (which includes the product version and build number, for example 11.3.1.6005) in the **Controlled Release** field just above the table of product releases, then click **Search** . 

The Hotfix release will display on the product matrix and can be selected for ordering and download. If you have questions about RBC, please contact us at U2Support@rocketsoftware.com. 

**Note:** Hotfix releases are intended to be short-term solutions and have had limited testing in order to be made available quickly. All Hotfix changes will be included in the next GA release, and will be fully tested by Rocket’s Quality Assurance (QA) team across all supported platforms and environments. It is strongly recommended that after taking a Hotfix customers upgrade to the next available GA release to ensure the highest quality experience. 

For questions, please contact support atU2Support@rocketsoftware.com .

## Rocket UniVerse Hotfix 12.1.1.1010 

## August 2019 

## Applicable platforms 

- Windows 

- Linux 

- AIX 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at support@rocketsoftware.com. 

## Description of the problem 

|**Issue**<br>**number**|**Description**|
|---|---|
|UNV-28152|Prior to this release, the colon (:) character in a BASIC CallHTTP request would be<br>encoded to '%3A'. This could cause failures when connecting to certain sites. In<br>the current release, the default behavior is to not encode the colon character. If<br>there is a requirement to encode the colon character, the environment variable<br>COLON_ENCODES can be set.|
|UNV-28185|Prior to this release, the behavior of the BASIC`GET`statement was inconsistent when<br>used with`TIMEOUT`on an NLS enabled system. The`GET`statement might timeout at<br>an interval different from that specified in the`TIMEOUT`statement. This issue has been<br>resolved.|
|UNV-28270|Windows only. Prior to this release, the BASIC`XDOMLocate`function could crash when<br>encoding was set to UTF-8 by the`XMLSETOPTIONS`command. This issue has been<br>resolved.|
|UNV-28313|Beginning with 12.1.1, the performance of the`BUILD.INDEX`command was<br>negatively impacted under certain circumstances. If the index being built resulted in<br>a very large number of keys being associated with an index value, the performance of<br>`BUILD.INDEX`was degraded from prior releases. This issue has been resolved.|

## Rocket UniVerse Hotfix 12.1.1.1009 

## July 2019 

## Applicable platforms 

## ▪ Windows 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at support@rocketsoftware.com. 

## Description of the problem 

|**Issue**<br>**number**|**Description**|
|---|---|
|UNV-28287|Windows only. Beginning with 12.1.1, problems could be encountered after running TCL<br>commands that start a child process. If the child process caused paging (Press any key<br>to continue...) to occur, executing other commands in the same session might behave<br>incorrectly. For example, executing another command where the child process resulted<br>in paging would cause the current telnet session to abort. Other incorrect behaviors<br>were also observed including problems with cursor keys, PORT.STATUS paging, and SB<br>file transfer functionality.<br>Note, the problems were only experienced when running a UniVerse telnet server<br>process. This issue has been resolved.|
