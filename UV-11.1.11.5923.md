## **Rocket UniVerse Hotfix** 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at U2Support@rocketsoftware.com. 

## **Version 11.1.11.5923** 

## **Applicable Platforms** 

- UniVerse 32-bit on Solaris Sparc 

## **Description of the problem** 

|**Issue number**|**Description**|
|---|---|
|UNV-15701|At previous releases, the fixtool utility may have reported an incorrect<br>record count. If the last record in a group buffer was in use, the record<br>might have been missed in the record count reported by fixtool. This<br>would result in a different record count reported by fixtool compared to<br>a TCL COUNT command. The fixtool utility will now report an accurate<br>record count. This issue has been resolved.|



## **Included in this Hotfix** 

The UV_SOLARIS_11.1.11.5923.zip file for this Hotfix contains the following: 

- fixtool 

## **Installation instructions** 

1. Extract the **fixtool** binary from the UV_SOLARIS_11.1.11.5923.zip file into a temporary location on the **SOLARIS** machine. 

2. Make a backup copy of the **fixtool** binary in uvhome/bin. For example, **cp fixtool fixtool.orig** . 

3. Copy the **fixtool** binary from the temporary location into uvhome/bin. 

4. Make sure the permissions on **fixtool** match the original file. For example, **chmod 751 fixtool** . 

You do not need to restart the UniVerse services after applying this Hotfix.
