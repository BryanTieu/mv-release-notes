## Rocket UniVerse Hotfix 11.3.0.5001 

## Version 11.3.0.5001 

## Applicable platforms 

- HP Itanium 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at U2Support@rocketsoftware.com. 

## Description of the problem 

|**Issue**<br>**number**|**Description**|
|---|---|
|UNV-23299|Beginning at 11.3.0, in certain situations the "Done" message displayed when a<br>phantom completes may have been included in the output captured via the EXECUTE/<br>CAPTURING statement. If NOTIFY ON or OFF were enabled and a BASIC program was<br>executing an operating system level command via "SH -c" while using the CAPTURING<br>clause, the "Done" message could have ended up in the capturing variable. This was<br>due to a timing issue related to having the child phantom process finish while the "SH<br>-c" command was processing. At the current release, the messages are suppressed in<br>this situation, resolving the problem.|
|UNV-23302|Prior to this release, certain BASIC socket and CallHTTP functions may have failed<br>on HP machines. The failures could appear random. The use of protocol logging<br>could also impact whether or not a failure would occur. The problem was caused by<br>incorrect parameter types being passed to certain system functions. This issue has<br>been resolved.|
|UNV-23449|Prior to this release, the EDA conversion process might have unexpectedly aborted if<br>the file being converted contained an I-type dictionary item. If using the EDA Schema<br>Manager tool, an RPC connection error would be displayed and the conversion process<br>would not complete. This issue has been resolved at the current release.<br>If this problem had been encountered at the previous release, we recommend that you<br>perform the following cleanup steps:<br>1.<br>Execute DELETE.FILE on the partially converted file.<br>2.<br>Remove the edaworking VOC entry.<br>3.<br>Remove the edasave and edaworking files at the OS level.<br>4.<br>Delete the schema map for the file.<br>5.<br>Restore the original UniVerse file to the account, and recreate the schema map.<br>Once this has been done, the conversion process can be executed again at the new<br>release.|
