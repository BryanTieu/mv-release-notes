## General Hotfix information 

This release note contains a cumulative summary of all the Hotfixes made available for a specific Generally Available (GA) product version. Please browse this document to find the Hotfix release that fits your specific needs. 

Hotfixes are identified as "Controlled Releases" within Rocket Business Connect (RBC). Hotfixes are only visible by searching for the specific version and build number in the Controlled Release field. 

To obtain a specific Hotfix: 

1. Log into the RBC system and modify your existing license. 

2. Click **Edit** next to your existing product and then click **Search** to view the list of GA product releases. 

3. After the product availability matrix displays, enter in the Hotfix version number (which includes the product version and build number, for example 11.3.1.6005) in the **Controlled Release** field just above the table of product releases, then click **Search** . 

The Hotfix release will display on the product matrix and can be selected for ordering and download. If you have questions about RBC, please contact us at U2Support@rocketsoftware.com. 

**Note:** Hotfix releases are intended to be short-term solutions and have had limited testing in order to be made available quickly. All Hotfix changes will be included in the next GA release, and will be fully tested by Rocket’s Quality Assurance (QA) team across all supported platforms and environments. It is strongly recommended that after taking a Hotfix customers upgrade to the next available GA release to ensure the highest quality experience. 

For questions, please contact support at U2Support@rocketsoftware.com .

## Rocket UniVerse Hotfix 11.3.2.7000.CF1 

## April 2020 

**Important: The 11.3.2.7000.CF1 release of UniVerse supports OpenSSL version 1.0.2u for FIPS compliance. The standard 11.3.2 version of UniVerse supports OpenSSL 1.1.1. Any new features in 11.3.2 related to OpenSSL 1.1.1 are not applicable to 11.3.2.7000.CF1.** 

## Applicable platforms 

- Linux 

- Windows 

- Solaris x86 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at support@rocketsoftware.com. 

## Description of the problem 

|**Issue**<br>**number**|**Description**|
|---|---|
|UNV-26152|Beginning with this release, the_ALLOWNULLS_uvconfig parameter has been added. The<br>_ALLOWNULLS_parameter can be used to control the ability to write null keys to UniVerse<br>files. Setting this parameter to 1 (default) allows null keys to be written. When set to 0,<br>attempting to`WRITE`a null key will cause a fatal error and terminate the program, or<br>take the`ON ERROR`clause if it exists.|
|UNV-28660|Prior to this release, executing a`LIST`statement might cause the UniVerse session to<br>exit unexpectedly under certain conditions. If the`LIST`statement included an I-type<br>subroutine, which increased the current terminal width`(!SETPU(PU$WIDTH))`<br>during execution, the UniVerse session might exit unexpectedly. This issue has been<br>resolved.|
|UNV-28669|Prior to this release, passing a string which exceeded the maximum length defined in<br>the GCI definition file could cause the uvsh process to terminate. In the current release,<br>the string is truncated to the maximum size and a warning message is displayed<br>indicating the string passed to the GCI routine was too large. This issue has been<br>resolved.|
|UNV-28918|Prior to this release, the`TRANS`function did not properly respect the rules of PI/OPEN<br>flavored accounts and would incorrectly lower characters 251 and below. This issue has<br>been resolved.|
|UNV-29079|Beginning with 11.3.1, a problem was found when using an SSH client to connect to a<br>UniVerse Windows server. If the SSH server delivered with Windows 10 was installed,<br>entering cursor keys which sent escape sequences would cause the UniVerse session to<br>exit. This issue has been resolved.|
|UNV-29090|The version of OpenSSL available with this release is 1.0.2u.|
|UNV-29101|Prior to this release, the BASIC`SELECTINDEX`statement would incorrectly return<br>a 0 key active select list when used with an alternate key value which did not exist.<br>The`SYSTEM(11)`function would indicate that a select list was active. In the current<br>release, an active select list is only returned when`SELECTINDEX`is used with an<br>existing alternate key value. This issue has been resolved.|

|UNV-29144|Beginning with this release, new BASIC compiler directives have been added to<br>correspond to the new BASIC functionality added at 11.3.2. The com-<br>piled object code will be placed in BP with the name`program.IRUN`. The`CATALOG`<br>command will catalog_`progname`_`.IRUN`if the object code for_progname_does not exist<br>in BP.O.|

Assigning an open sequential file variable to another<br>variable and using that variable with other BASIC sequential I/O statements such as<br>`WRITESEQ`would cause the UniVerse session to terminate. Other conditions using<br>sequential I/O could also result in session termination. This issue has been resolved.|
|UNV-29188|Prior to this release, if a fatal error was encountered when executing`SET.FIPS.MODE`<br>or`GET.FIPS.MODE`, the error would incorrectly be displayed when the`HUSH`or<br>`BRIEF`keywords were used. In the current release, all output generated by these<br>commands will be suppressed when the`HUSH`or`BRIEF`keywords are used.|

Executing<br>the`uvaudd`command as defined in the`uv.rc`script or rebooting the server was<br>required to restart Audit. This issue has been resolved.|
|UNV-29230|Prior to this release, the UniVerse session would terminate when using the BASIC<br>`FILEINFO()`function to return the path of a file defined with a path of "`..`". This<br>issue has been resolved.|
