## General Hotfix information 

This release notes document contains a cumulative summary of all the Hotfixes made available for a specific Generally Available (GA) product version. Please browse this document to find the Hotfix release that fits your specific needs. 

Hotfixes are identified as "Controlled Releases" within Rocket Business Connect (RBC). Hotfixes are only visible by searching for the specific version and build number in the Controlled Release field. 

To obtain a specific Hotfix: 

1. Log into the RBC system and modify your existing license. 

2. Click **Edit** next to your existing product, and then click **Search** to view the list of GA product releases. 

3. After the product availability matrix displays, enter the Hotfix version number (which includes the product version and build number, for example 11.3.1.6005) in the **Controlled Release** field just above the table of product releases. 

4. Click **Search** . 

The Hotfix release will display on the product matrix, and can be selected for ordering and download. If you have questions about RBC, please contact us at U2Support@rocketsoftware.com. 

**Note:** Hotfix releases are intended to be short-term solutions and have had limited testing in order to be made available quickly. All Hotfix changes will be included in the next GA release, and will be fully tested by Rocket’s Quality Assurance (QA) team across all supported platforms and environments. It is strongly recommended that after taking a Hotfix, customers upgrade to the next available GA release to ensure the highest quality experience. 

For questions, please contact support at U2Support@rocketsoftware.com.

## Rocket UniVerse Hotfix 11.3.4.9004 

## April 2022 

## Applicable platforms 

- AIX 

- Linux 

- Windows 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at support@rocketsoftware.com. 

## Description of the problem 

|**Issue**<br>**number**|**Description**|
|---|---|
|UNV-32381|Prior to this release, the internal performance counter that records the number of times<br>temporary files are closed always returned 0. This issue has been resolved.|
|UNV-32414|Beginning with this release, several issues related to the monitoring of file usage<br>statistics by the MVX performance monitoring functionality have been resolved. These<br>include corruption in the shared memory used to record statistics, not being able to<br>disable the monitoring, and accessing the shared memory before fully initialized. These<br>issues have been resolved.|
|UNV-32426|Beginning with this release, the GET.RESIZE.PARAMS utility has been modified so that it<br>bypasses the LOGIN proc or paragraph in the UniVerse account being used. This avoids<br>the possibility that the command will fail due to the contents of the LOGIN proc or<br>paragraph.|
|UNV-32440|Prior to this release, the user process information displayed by MVX might have<br>been incorrect in certain situations. If the UniVerse process command line contained<br>characters illegal to JSON, the parsing of the command line would result in errors. This<br>issue has been resolved.|
|UNV-32484|Prior to this release, replicating sequential I/O might have performed incorrectly under<br>certain conditions. If the directory name containing the sequential file was larger than<br>128 characters, a WRITESEQ statement to the sequential file may have hung. This issue<br>has been resolved.|

Install as a typical installation. Refer to the _UniVerse Installation_ manual for details.

|---|---|
|UNV-31915|Prior to this release, when FIPS mode was enabled, using an unsupported algorithm<br>with the BASIC DIGEST() function would cause the UniVerse process to terminate. The<br>DIGEST() function now returns an error code indicating an unsupported algorithm,<br>resolving this issue.|

The missing character would cause the decryption of the record to fail on<br>the subscriber. This issue has been resolved.|
