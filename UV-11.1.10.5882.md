## Rocket UniVerse Hotfix 11.1.10.5882 

## Version 11.1.10.5882 

## Applicable platforms 

- AIX 

This download contains the most recent Hotfix for the previously released product. For questions, please contact support at U2Support@rocketsoftware.com. 

## Description of the problem 

|**Issue**<br>**number**|**Description**|
|---|---|
|UNV-23450|Prior to this release, the memory allocation process in the SIGCHLD handler caused<br>storage corruption. This issue has been resolved.|
|UNV-20920;<br>UNV-21020|Prior to this release, when a UniVerse phantom process forked an external process,<br>the pid of the child process was not cleaned up correctly from the internal job queue<br>when the child exited. This behavior could have resulted in subsequent child processes<br>hanging if the pid of the new process matched a pid that was incorrectly left in the job<br>queue. This issue has been resolved.|



## Included in this release 

The UV_AIX_11.1.10.5882.zip file for this Hotfix contains the following files: 

- uvsh 

## Installation instructions 

1. Stop UniVerse or at a minimum, stop all running uvsh processes. 

2. **Optional** . Make a backup copy of the existing uvsh. 

3. Copy uvsh to your UniVerse bin directory. For example /usr/uv/bin. 

4. Restart UniVerse.
